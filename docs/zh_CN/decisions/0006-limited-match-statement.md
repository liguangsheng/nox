# 0006 - 受限 match 语句

- 状态：已采纳
- 日期：2026-05-20
- 涉及：语言

## 背景

Nox 已有 `if` / `else if`，但按整数或字符串常量分派时会写出重复条件。v0.0.2 需要
一个结构化多分支语句，同时避免引入完整模式匹配、guard、值绑定或新的 enum 类型。

## 决策

引入语句式 `match`：

```nox
match (value) {
    1 => {
        ...
    }
    _ => {
        ...
    }
}
```

匹配值必须是 `int` 或 `str`。case 头只能是同类型字面量；`_` 默认分支必填。重复
case 是静态错误。每个分支 body 是独立块，没有 fallthrough。第一批不支持表达式形式
`let x = match (...) { ... }`。

编译上不新增 VM 指令：compiler 把 `match` 降成等值比较和 `JumpIfFalse` 链。为避免
重复求值，匹配表达式会先存入编译器生成的临时绑定。

## 后果

脚本获得了清晰的常量分派语法，type checker 可以静态拒绝缺少默认分支、重复 case、
case 类型不匹配和不支持的匹配值类型。语义保持简单；未来如果要支持表达式形式或更完整
模式匹配，需要单独设计返回类型合流、穷尽性和绑定作用域。

## 备选方案

- 继续只用 `if` / `else if`：被拒绝，常量分派可读性差。
- C 风格 `switch` / `case`：被拒绝，容易引入 fallthrough 语义。
- Rust 风格完整模式匹配：暂不选择，v0.0.2 没有 enum、destructuring 或 guard 设计。
